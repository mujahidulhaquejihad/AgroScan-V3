  AGROVET V2 — HOW THE WEB APP WORKS (BEGINNER-FRIENDLY GUIDE)
================================================================================

This document explains the whole project in simple words. You do not need to
be a programmer to understand the big picture. Technical terms are explained
when they appear.


--------------------------------------------------------------------------------
1. WHAT IS THIS APP?
--------------------------------------------------------------------------------

AgroVet V2 is a website that helps farmers and gardeners check plant leaf
health from a photo.

You upload (or take) a picture of a leaf. The app tries to answer:

  • Is this actually a leaf? (not a random photo of a shoe, wall, etc.)
  • If yes, what plant is it and is the leaf healthy or diseased?
  • What treatment and prevention advice might help?

The app is built for Bangladesh users:
  • Default language is Bengali (Bangla)
  • Emergency numbers like Krishi 16123 and Vet 16358 are built in
  • Government agriculture links are listed in the Resources section

Important: This is a helper tool, not a replacement for a real plant doctor
or veterinarian. Always call official helplines for serious cases.


--------------------------------------------------------------------------------
2. THE BIG PICTURE (ONE SENTENCE)
--------------------------------------------------------------------------------

Your phone or computer shows a web page (the "frontend"). That page talks to
a Python program running on your PC (the "backend"). The backend loads AI
models, looks at your photo, and sends back results as JSON (structured text
data).


--------------------------------------------------------------------------------
3. HOW YOU START THE APP
--------------------------------------------------------------------------------

Step 1 — Open a terminal in the project folder:
         I:\Agrovet V2

Step 2 — Run this command:

         python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000

         What this means:
         • python          = runs Python code
         • uvicorn         = a small web server program
         • backend.main:app = the main server file (backend/main.py)
         • --host 0.0.0.0  = allows other devices on your Wi-Fi to connect
         • --port 8000     = the app listens on port 8000

Step 3 — Open a browser and go to:

         http://localhost:8000

         Do NOT double-click index.html directly. The page must be served by
         the Python server so the API calls work.

When the server starts, it:
  1. Creates/opens a small user database (SQLite)
  2. Loads all AI model files from the models/ folder into memory
  3. Prints "Engine status" in the terminal so you can see what loaded

5. FRONTEND — WHAT EACH PIECE DOES
--------------------------------------------------------------------------------

The browser loads one HTML page (index.html) plus several JavaScript files.

HTML (index.html)
  • Structure of the page: header, upload area, result cards, sidebars,
    emergency section, government resources, chat button, footer.

CSS (styles.css)
  • Colors, layout, mobile bottom navigation, dark mode styling.

JavaScript files (loaded in order):

  config.js
    • Sets API_BASE (where to send requests — same server as the page).
    • Holds Google Sign-In client ID (optional).

  i18n.js
    • Language system: Bengali ↔ English toggle.
    • All button labels and messages can switch language.

  auth.js
    • Login / signup / logout UI in the header.
    • Saves session token in browser localStorage.

  core.js  ← MOST IMPORTANT FOR DIAGNOSIS
    • Checks if server is online (/api/status).
    • Handles file pick, camera, drag-and-drop.
    • Compresses large photos before upload.
    • Sends image to /api/predict.
    • Draws results on screen (leaf check, disease, advice).

  features.js
    • Dark mode, diagnosis history (localStorage).
    • Weather + spraying advice (Open-Meteo API).
    • Crop calendar, daily tip, disease library search.
    • Text-to-speech and download report.

  app-extra.js
    • Government links grid, emergency cards, chatbot popup.
    • Drag-and-drop on upload zone.

  mobile.js
    • Bottom navigation on phones, PWA install hints.

Auth pages (separate HTML):
  login.html, signup.html, logout.html — same styling, simpler pages.

6. BACKEND — THE API SERVER
--------------------------------------------------------------------------------

File: backend/main.py
Framework: FastAPI (a modern Python web framework)

When you visit http://localhost:8000/ the server sends index.html.
Static files (CSS, JS, images) are served from /static/... which maps to
the web/ folder.

Main API endpoints (URLs the frontend calls):

  GET  /api/status
       → Is the server ready? Which models are loaded? CPU or GPU?

  GET  /api/classes
       → List of all disease class names the models know.

  POST /api/predict
       → Upload an image file → get diagnosis JSON back.

  POST /api/chat
       → Send a text message → get a simple rule-based reply.

  GET  /api/resources
       → Emergency phone numbers + government website links.

  GET  /api/advice/{disease_name}
       → Treatment/prevention text for one disease.

  GET  /api/diseases
       → Full disease encyclopedia for the library sidebar.

  POST /api/auth/signup, /api/auth/login, /api/auth/logout
  GET  /api/auth/me
  POST /api/auth/google
       → User accounts (optional — you can use the app as guest too).

7. THE AI MODELS — EXPLAINED SIMPLY
--------------------------------------------------------------------------------

What is an AI model here?
  A model is a file (`.pt` = PyTorch checkpoint) that was TRAINED on thousands
  of labeled photos. It learned patterns like "this tomato leaf has late blight"
  vs "this tomato leaf is healthy."

Your project uses PyTorch + torchvision. Each model is a famous neural network
architecture (think: a recipe for how to process images) with the last layer
changed to output your plant disease categories.

Model files in models/ folder:

  leaf_gate.pt                  (~17 MB)
  disease_efficientnet_b3.pt    (~44 MB)
  disease_resnet50.pt           (~95 MB)
  disease_mobilenet_v3_large.pt (~17 MB)


--- STAGE 1: LEAF GATE (safety check first) ---

Model: MobileNet V3 Large (fast, lightweight)
File:  models/leaf_gate.pt

Purpose:
  Before guessing a disease, ask: "Is this photo actually a plant leaf?"

Classes: basically "leaf" vs "non_leaf"

How it decides:
  • Runs the image through the model
  • Gets a probability (0% to 100%) that it is a leaf
  • If probability >= 50% → treat as leaf and continue
  • If below 50% → STOP. No disease analysis. User sees a message in Bengali.

Why this matters:
  Stops the app from confidently labeling a photo of a hand or table as
  "Tomato Late Blight."


--- STAGE 2: DISEASE DETECTION (three models working together) ---

If Stage 1 says "yes, this is a leaf," three separate models all analyze the
same image:

  1. EfficientNet-B3   (file: disease_efficientnet_b3.pt)
  2. ResNet-50         (file: disease_resnet50.pt)
  3. MobileNetV3       (file: disease_mobilenet_v3_large.pt)

Each model was trained on the same dataset:
  PlantVillage-style augmented dataset — about 38 disease classes covering
  plants like Tomato, Potato, Corn, Apple, Grape, Pepper, etc.

Example class name format:
  Tomato___Late_blight
  (Plant name) ___ (condition)

What each model outputs:
  • Its best guess (class name)
  • Confidence score (how sure it is, 0 to 1)
  • Top 3 guesses with scores

ENSEMBLE = "BEST ANSWER"
  The app does NOT just pick one model. It averages the probability scores
  from all three models and picks the class with the highest average.

  It also counts agreement: "2/3 models agree" or "3/3 models agree."

Confidence warnings:
  • Below 45% → marked "uncertain"
  • Below 80% → user is warned to retake photo or call 16123 / 16358

After picking the best disease class, the server attaches written advice
(treatment + prevention) from the knowledge base (see section 9).


--- WHERE DOES INFERENCE CODE LIVE? ---

  agrovet/infer.py  → class InferenceEngine
    • Loads models on startup
    • predict(image) runs Stage 1 then Stage 2
    • Returns one big JSON result

  agrovet/models.py → builds neural networks, loads .pt files
  agrovet/config.py → paths, model names, thresholds
  agrovet/data.py   → image resizing/normalization for inference


--- CPU vs GPU ---

  On startup the engine picks:
    • CUDA (NVIDIA GPU) if available — faster
    • otherwise CPU — slower but works on any PC

--------------------------------------------------------------------------------

  [YOU]  Pick image or take photo
           │
           ▼

                        │

                resnet50         ──┼──► average probabilities ──► best_answer

                        │

           │

9. KNOWLEDGE BASE & CHATBOT (NOT ChatGPT)
--------------------------------------------------------------------------------

Files:
  agrovet/knowledge.py      — English + shared data
  agrovet/knowledge_bn.py   — Bengali disease advice text

The treatment advice under results is NOT generated by an LLM. It is pre-written
text stored in Python dictionaries, keyed by disease class name.

The chatbot (/api/chat) is also rule-based:
  • Looks for keywords in your message (disease names, "fungus", "16123", etc.)
  • Picks a canned helpful reply
  • Can use context_disease if you just ran a diagnosis
  • Responds in Bengali or English based on lang parameter

Government links and emergency contacts are hard-coded lists in knowledge.py,
loaded by /api/resources and displayed in the Resources and Emergency sections.


--------------------------------------------------------------------------------
10. USER ACCOUNTS & DATA STORAGE
--------------------------------------------------------------------------------

Authentication is OPTIONAL. You can browse and diagnose without logging in.

If you sign up / log in:

  Database: data/users.db (SQLite — a single file database)
  Code:     agrovet/auth_store.py

  Tables:
    users    — name, email, password hash, Google info
    sessions — login tokens with expiry (30 days)

  Passwords are NOT stored plain text. They use PBKDF2 hashing with salt.

  After login, the browser stores a token in localStorage (key: agrovet_session).
  Future API calls can send: Authorization: Bearer <token>

What is NOT in the database:
  • Your uploaded leaf photos (not saved on server by default)
  • Diagnosis history (saved only in YOUR browser localStorage)
  • AI model weights (separate .pt files in models/)

12. HOW THE MODELS WERE TRAINED (SHORT VERSION)
--------------------------------------------------------------------------------

Training scripts (run separately, not when using the web app):

  agrovet/train_leaf.py    → trains leaf_gate.pt
  agrovet/train_disease.py → trains one disease_*.pt per architecture
  agrovet/train_all.py     → runs the full training pipeline

Training data folders (under Datasets/):
  • Leaf vs non-leaf images for the gate model
  • PlantVillage augmented train/valid folders for disease models

During training:
  • Images are augmented (flips, crops, color jitter) to improve robustness
  • Models start from ImageNet pretrained weights (general vision knowledge)
  • Fine-tuned on plant disease photos
  • Best weights saved to models/*.pt

The web app only LOADS these finished files — it does not train during use.

15. COMMON BEGINNER QUESTIONS
--------------------------------------------------------------------------------

Q: Why does the status pill say "connecting" or "offline"?
A: The browser cannot reach http://localhost:8000/api/status. Start the server
   with the uvicorn command. Do not open the HTML file directly.

Q: Why does it say models not trained?
A: The .pt files are missing from models/ folder. Train them or copy trained
   checkpoints into that folder.

Q: Does my photo get sent to the internet?
A: By default everything runs locally on your PC. The image goes from browser
   to your own Python server. Exception: weather widget calls Open-Meteo;
   Google login calls Google if you use that feature.

Q: Why three models instead of one?
A: Different architectures make different mistakes. Averaging them often gives
   more reliable answers than trusting a single model.

Q: What if I upload not-a-leaf?
A: Stage 1 blocks disease analysis and shows a Bengali skip message.

Q: Bengali is default — how does English work?
A: Click the language button in the header. i18n.js swaps UI strings; advice
   and chat can request English via lang parameter.

16. GLOSSARY
--------------------------------------------------------------------------------

  API          — Application Programming Interface; URLs the frontend calls
  FastAPI      — Python library that creates the web server
  PyTorch      — Python deep learning framework (.pt files)
  Inference    — Using a trained model to predict (not training)
  Ensemble     — Combining multiple models for one answer
  Softmax      — Converts model outputs into percentages that sum to 100%
  JSON         — Text format for structured data { "key": "value" }
  SQLite       — Small file-based database (users.db)
  localStorage — Data saved inside your browser only
  Frontend     — HTML/CSS/JavaScript in the browser
  Backend      — Python server + AI models on your machine
  Uvicorn      — Program that runs the FastAPI app
